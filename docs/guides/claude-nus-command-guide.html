# 01

Cái bẫy: đừng để dư /v1 trong base URL

Trang key đưa endpoint dạng https://llm.dev.nustechnology.com/v1 — đúng cho config kiểu OpenAI-compatible. Nhưng Claude Code tự nối thêm /v1/messages vào ANTHROPIC_BASE_URL nó nhận. Giữ nguyên /v1 ở cuối thì request thật sự bay tới /v1/v1/messages → 404, và Claude Code báo nhầm “model không tồn tại” — sai hoàn toàn, lỗi thật là path bị lặp. Bỏ /v1 cuối là dùng được, vì endpoint này nói được luôn chuẩn Anthropic Messages API song song với chuẩn OpenAI.

# 02

File key, quyền riêng tư

Không paste key thẳng vào .bashrc hay commit vào repo. Tạo 1 file riêng, quyền chỉ chủ sở hữu đọc được — claude-nus sẽ đọc từ đây:

terminal
mkdir -p ~/.config/nus-llm
umask 077
cat > ~/.config/nus-llm/env << 'EOF'
NUS_LLM_BASE_URL=https://llm.dev.nustechnology.com
NUS_LLM_API_KEY=YOUR_KEY
NUS_LLM_MODEL=deepseek-v4-flash
EOF
chmod 600 ~/.config/nus-llm/env

Model khả dụng: deepseek-v4-flash (nhanh, rẻ, mặc định) và deepseek-v4-pro (chất lượng cao hơn).

# 03

Viết script

Tạo file thực thi, không cần đuôi mở rộng — đây là 1 lệnh CLI, không phải script gọi qua tên khác:

~/.local/bin/claude-nus
#!/usr/bin/env bash
# Explicit: force Claude Code onto the NUS company DeepSeek key, no probing,
# no fallback. Use when you deliberately want NUS regardless of subscription
# quota (e.g. to save subscription usage on throwaway tasks).
set -uo pipefail

ENV_FILE="$HOME/.config/nus-llm/env"
if [[ ! -f "$ENV_FILE" ]]; then
  echo "claude-nus: $ENV_FILE not found -- run the setup steps first" >&2
  exit 1
fi

# shellcheck source=/dev/null
source "$ENV_FILE"
MODEL="${NUS_MODEL:-$NUS_LLM_MODEL}"

echo "[claude-nus] using NUS LLM ($MODEL)" >&2
exec env -u ANTHROPIC_API_KEY -u ANTHROPIC_AUTH_TOKEN \
  ANTHROPIC_BASE_URL="$NUS_LLM_BASE_URL" \
  ANTHROPIC_API_KEY="$NUS_LLM_API_KEY" \
  claude --model "$MODEL" "$@"

Vì sao env ... claude mà không đệ quy lại chính claude-nus? env là 1 binary ngoài, nó tự tìm claude qua PATH của tiến trình con — không biết gì về function/alias của shell cha, nên luôn trúng đúng binary Claude Code thật, kể cả khi bạn đã override claude bằng function khác trong .bashrc.

# 04

Cấp quyền thực thi

terminal
chmod +x ~/.local/bin/claude-nus

Kiểm tra ~/.local/bin có trong PATH chưa (thường có sẵn trên máy đã cài Claude Code, vì bản thân claude cũng nằm ở đó):

terminal
echo "$PATH" | tr ':' '\n' | grep local/bin

Không thấy thì thêm vào cuối ~/.bashrc: export PATH="$HOME/.local/bin:$PATH"

# 05

Chạy thử

terminal
claude-nus -p "Reply with exactly: OK WORKS"

Kỳ vọng: stderr in ra [claude-nus] using NUS LLM (deepseek-v4-flash), rồi OK WORKS ở stdout. Không cần tham số gì thêm là vào chế độ chat interactive như claude thường, chỉ khác endpoint.
