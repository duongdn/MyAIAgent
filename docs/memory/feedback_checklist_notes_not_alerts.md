---
name: Trello checklist item notes are NOT alerts
description: Text in checklist item names (risk warnings, reminders, task descriptions) are the user's personal notes — do NOT treat them as alerts blocking completion
type: feedback
---

Trello checklist item names often contain the user's personal notes/reminders after the project name, e.g.:
- "Aysar - review, đào task - đang rất RISK - HẾT TASK"
- "Elliott - GreenFort Capital - performance issue (pending)"
- "Marcel - update deploy with permission - monitor storage"

These notes are the user's reminders for themselves. Do NOT treat them as alerts that block completion.

**Why:** User said "Please never read the checklist item contain as risk, this is my note only."
**How to apply:** Only use MONITORING DATA (Slack, Email, Discord, task logs, etc.) to determine alerts. Ignore text in the checklist item name when deciding whether to complete. The item name is just a label + user notes.
